Etapa 2 — Estrutura operacional da folha

Horários, centros de custo, bancos e departamentos · ETA: 15–20 min

URL base: #/wizard/etapa-2/ · Código: frontend/src/pages/wizard/stage2.tsx

Objetivo: cadastrar as estruturas que serão usadas em vínculos, rateios e pagamentos — com opção de importação CSV em horários e centros de custo. Depende da etapa 1 aprovada.

passo-1 — Entrada da etapa

Tela informativa: não há formulário de persistência próprio; mostra dependências e quantidades já cadastradas.

Elemento Fonte Para que serve

passo-2 — Relação de horários

Modo manual (modal) ou importação CSV. Tabela lista horários existentes com exclusão por linha.

Campo Obrig. Para que serve Como funciona
Modo de entrada — Escolher cadastro manual ou CSV. Radio manual / import. CSV: textarea + botão “Importar horários” → POST /work-schedules/import-csv.
CSV (textarea) Se import Carga em lote. Formato: nome;journeyType;entrada;saida;intervaloMin;dias — dias como 1,2,3,4,5; tipo FIXED ou SHIFT.
name Sim (manual) Nome do horário (ex.: Administrativo). POST /work-schedules
journeyType Sim Fixa ou escala. FIXED = mesma escala recorrente; SHIFT = revezamento de turnos.
startTime / endTime Não* Entrada e saída (HH:mm 24h). *Validações legais aparecem como alertas quando preenchidos com intervalo.
breakMinutes Não Intervalo intrajornada em minutos. Padrão inicial 60 no modal; mínimo 0.
weekdays Não Dias da semana da jornada. Checkbox group: 0=Dom … 6=Sáb. Padrão Seg–Sex.

Colunas da tabela: Nome, Jornada, Entrada, Saída, Intervalo (min), Dias, Excluir.

passo-3 — Centros de custo

Campo Obrig. Para que serve Como funciona
Modo manual / CSV — Fluxo de cadastro. CSV: codigo;nome;cnpjFilialOpcional (somente dígitos no CNPJ da filial) → POST /cost-centers/import-csv.
code Sim Código interno (integração, relatórios). POST /cost-centers com active: true.
name Sim Nome exibido na operação. —
branchId Não Vincular CC a uma filial. Select das filiais da etapa 1; vazio = matriz.

Tabela: Código, Nome, Filial (nome ou “Matriz”).

passo-4 — Bancos e agências

Lista FEBRABAN carregada de /banks. Valida agência com regras BR razoáveis.

Campo Obrig. Para que serve Como funciona
bankReferenceId Sim Banco para pagamento de salários. Select pesquisável: código + nome.
agency Sim Número da agência (1 a 5 dígitos). Validação isReasonableBankAgency.
agencyDigit Não DV da agência quando existir. Até 2 caracteres numéricos.
accountType Sim Tipo de conta para crédito. CHECKING (corrente), SAVINGS (poupança), PAYMENT (conta pagamento). Padrão: corrente.

Tabela: Banco (código — nome), Agência, Tipo conta.

passo-5 — Departamentos

Campo Obrig. Para que serve Como funciona
code Sim Código interno (RH, OP, etc.). POST /departments com active: true.
name Sim Nome em holerites e relatórios. —
costCenterId Não Departamento ligado a um CC. Select dos centros de custo; pode limpar.

Tabela: Código, Nome, CC (código ou —).
