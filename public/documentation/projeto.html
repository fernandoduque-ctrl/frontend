Visão do projeto Apêndice técnico

Esta página é para desenvolvedores e arquitetos. Para narrativa de negócio, use O que vamos entregar ou Início.

O frontend é um assistente de parametrização de folha para o produto Folha Digital Sólides: conduz o cliente por etapas (empresa, estrutura operacional, base histórica, benefícios, rubricas e importação eSocial), com textos e checklists alinhados ao processo consultivo.

Stack técnica

  • React 18 + TypeScript
  • Vite como bundler e dev server
  • React Router em modo HashRouter (URLs com #)
  • Ant Design (antd) para UI
  • React Query (@tanstack/react-query) para dados remotos
  • React Hook Form + Zod onde há formulários validados
  • Axios via serviço de API

Pastas relevantes (src/)

  • pages/ — páginas por rota; pages/wizard/ contém stage1.tsx … stage6.tsx, StepContent.tsx, StepOutlet.tsx.
  • layouts/ — AppShell (menu lateral + conteúdo), WizardStageLayout (cabeçalho da etapa, tempo estimado).
  • components/ — ProtectedRoute, componentes reutilizáveis do wizard.
  • constants/ — stageMeta.ts (títulos, subtítulos, lista de passos por etapa).
  • utils/ — wizardSteps.ts, brForm.ts (validações BR), etc.
  • services/ — autenticação e cliente HTTP.

Modelo de dados (banco)

Diagrama entidade-relacionamento (Mermaid) e tabelas com colunas e chaves, gerados a partir do schema Prisma via API. Entrada em HTML estático: diagrama-banco.html (iframe da rota pública #/documentacao/diagrama-banco) ou demonstração online.

Autenticação

Rotas dentro de ProtectedRoute exigem sessão válida (JWT). A página de login é pública. Detalhes de rotas em Rotas e shell.
