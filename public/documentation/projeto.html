Visão do projeto Apêndice técnico

Esta página é para desenvolvedores e arquitetos. Para narrativa de negócio, use O que vamos entregar ou Início.

O frontend é um assistente de parametrização de folha para o produto Folha Digital Sólides: conduz o cliente por etapas (empresa, estrutura operacional, base histórica, benefícios, rubricas e importação eSocial), com textos e checklists alinhados ao processo consultivo.

Stack técnica (visão completa)

Versões alinhadas aos package.json dos repositórios frontend e backend.

Frontend (folha-solid-wizard-frontend)

  • React 18 + TypeScript (~5.6)
  • Vite 6 como bundler e dev server (@vitejs/plugin-react)
  • React Router DOM 6 em modo HashRouter (URLs com #)
  • Ant Design 5 (antd) + @ant-design/icons
  • TanStack React Query 5 para cache e chamadas à API
  • React Hook Form 7 + Zod 3 + @hookform/resolvers em formulários validados
  • Axios centralizado em src/services/api.ts (base URL via VITE_API_URL)
  • dayjs (datas, alinhado ao ecossistema Ant Design)
  • Mermaid 11 — diagrama ER na página pública #/documentacao/diagrama-banco (render + export SVG)
  • html2canvas + jspdf — geração de PDF onde usado no app
  • Script npm run lint no package.json (ESLint em src/**/*.{ts,tsx}); o pacote ESLint pode precisar ser adicionado às devDependencies para rodar em máquina limpa.

Backend (folha-solid-wizard-backend)

  • Node.js ≥ 20
  • NestJS 10 (HTTP com @nestjs/platform-express)
  • TypeScript (~5.7)
  • Prisma 5 — ORM, migrations e prisma/seed.ts
  • @nestjs/config — variáveis de ambiente (.env)
  • Autenticação JWT — @nestjs/jwt, @nestjs/passport, passport-jwt, senhas com bcrypt
  • Validação — class-validator + class-transformer + ValidationPipe global
  • OpenAPI / Swagger — @nestjs/swagger; UI em /api/docs
  • Uploads — multer; arquivos servidos como estáticos sob /files/ (pasta STORAGE_PATH ou storage/ no projeto)
  • Resposta da API — envelope padronizado via interceptor (TransformInterceptor) e filtro global de exceções
  • ESLint 9 + TypeScript ESLint 8 — script npm run lint

Banco de dados

  • SQLite no schema.prisma atual (DATABASE_URL) — adequado a dev/MVP local.
  • O schema é portátil: comentário no Prisma indica troca de provider e URL para PostgreSQL, MySQL ou SQL Server em outros ambientes.
  • Enums de domínio no SQLite ficam como String; constantes espelhadas em src/common/prisma-enums.ts no backend.

Integração front ↔ back

  • REST JSON; cabeçalhos Authorization: Bearer … e X-Company-Id quando aplicável (multi-empresa).
  • Contratos descritos no Swagger; página estática swagger.html embute a UI OpenAPI.

Documentação estática (este site)

  • HTML + CSS em public/documentation/; sem framework no guia.
  • Diagrama ER e tabelas do schema via API do backend na rota pública do app React.

Pastas relevantes (src/)

  • pages/ — páginas por rota; pages/wizard/ contém WizardEmpresaCadastro.tsx, WizardFolhaOperacional.tsx, WizardHistoricoTrabalhadores.tsx, WizardBeneficios.tsx, WizardRubricasEventos.tsx, WizardImportacaoEsocial.tsx, StepContent.tsx, StepOutlet.tsx.
  • layouts/ — AppShell (menu lateral + conteúdo), WizardStageLayout (cabeçalho da etapa, tempo estimado).
  • components/ — ProtectedRoute, componentes reutilizáveis do wizard.
  • constants/ — wizardEtapaMeta.ts (títulos, subtítulos, lista de passos por etapa).
  • utils/ — wizardSteps.ts, brForm.ts (validações BR), etc.
  • services/ — autenticação e cliente HTTP.

Modelo de dados (banco)

Diagrama entidade-relacionamento (Mermaid) e tabelas com colunas e chaves, gerados a partir do schema Prisma via API. Entrada em HTML estático: diagrama-banco.html (iframe da rota pública #/documentacao/diagrama-banco) ou demonstração online.

Autenticação

Rotas dentro de ProtectedRoute exigem sessão válida (JWT). A página de login é pública. Detalhes de rotas em Rotas e shell.
