Etapa 6 — Importação histórica eSocial

Staging, pré-visualização e confirmação · ETA: 30–45 min

URL base: #/wizard/etapa-6/ · Código: frontend/src/pages/wizard/stage6.tsx

Objetivo: simular (neste MVP) o fluxo completo de coleta governamental — certificado, período, eventos, processamento em lote, pré-visualização, confirmação explícita e acompanhamento. O acesso é bloqueado pelo backend quando faltam pré-requisitos (GET /wizard/stage-6/access), exceto no passo 1 informativo.

Tela adicional

Rota: #/wizard/etapa-6/lote/:batchId · Página EsocialLotePage.tsx — detalhe do lote fora do fluxo linear.

passo-1 — Entrada

Explica o escopo da importação (vínculos S-2200/S-2300, remunerações S-1200, etc.) e lista bloqueios quando access.allowed === false.

Sem campos de formulário; botão “Iniciar importação” só habilita se o acesso for permitido.

passo-2 — Certificado digital

Estado em memória (React) usado no corpo do lote no passo 5 — não é um Form.Item nomeado, mas são os parâmetros enviados à API.

Campo (estado) Opções / formato Para que serve Como funciona
certificateType (cert.type) A1 | A3 Tipo de certificado. Radio; A1 com upload .pfx (simulado), A3 token/cartão (simulado).
certificateTaxId (cert.cnpj) Texto (CNPJ) CNPJ vinculado ao certificado. Input texto; em produção deve bater com o da empresa.
environment (cert.env) PRODUCTION | RESTRICTED_PRODUCTION Ambiente eSocial. Select: Produção ou Produção restrita.
Senha do certificado — Autenticação na operação. Input.Password; mensagem na UI: não persistida de forma durável neste MVP.

Dica de teste na UI: fluxo com tipo A1_EXPIRED_SIM para simular bloqueio por validade (quando implementado no backend).

passo-3 — Período da importação

Campo Para que serve Como funciona
Data inicial (period.start) Início do histórico a buscar. DatePicker → ISO em periodStart no lote.
Data final (period.end) Fim do histórico. Alerta se posterior a “hoje”; recomenda-se até competência atual.
Amplitude (meses) Transparência de esforço. Calculada na UI; alerta se > 36 meses.
Validação de ordem Evitar intervalo inválido. Erro se início > fim.

passo-4 — Eventos selecionados

Valores enviados em selectedEvents (sempre inclui obrigatórios pelo front).

Valor enviado Rótulo na UI Obrigatório
trabalhadores Cadastros de trabalhadores (vínculos) Sim (checkbox desabilitado)
remuneracoes Remunerações Sim
pagamentos Pagamentos Não
afastamentos Afastamentos Não
ferias Férias Não
desligamentos Desligamentos Não
tabelas Tabelas vigentes (quando aplicável) Não

passo-5 — Processamento

Monta o corpo do lote e dispara o pipeline simulado.

Campo no JSON Origem na UI
certificateType Passo 2
certificateTaxId Passo 2 (opcional no payload se vazio)
environment Passo 2
periodStart / periodEnd Passo 3 (toISOString())
selectedEvents Passo 4 + obrigatórios

Sequência: POST /esocial-import/batches → guarda id no sessionStorage → POST /esocial-import/batches/:id/process.

passo-6 — Pré-visualização

Consome GET .../preview e GET .../alerts usando o ID do lote salvo localmente.

Elemento Descrição
Resumo numérico preview.counts.trabalhadores, remuneracoes, feriasAfastamentos
Aba Trabalhadores Colunas: recordType, externalId, payloadJson
Aba Remunerações Mesmas colunas
Aba Férias e afastamentos Mesmas colunas
Aba Alertas severity, title, description
Baixar log Download autenticado /esocial-import/batches/:id/export-log

passo-7 — Confirmação

Campo / ação Para que serve Como funciona
Checkbox de ciência Confirmação explícita do cliente. Obrigatório para habilitar o botão de confirmar.
Corpo da API Registro formal da aceitação. POST /esocial-import/batches/:id/confirm com confirmationTextAccepted: true.

passo-8 — Status do lote

Dado Origem
status GET /esocial-import/batches/:id
Link relatório completo Navega para #/wizard/etapa-6/lote/:batchId
Tabela comparativa (UX) “Abordagem comum” × “Folha Digital Sólides” — apenas informativo

Estados citados na UI: importando, importação concluída, concluída com alertas, erro (com log).

No MVP a gravação definitiva é simulada; o desenho de produção mantém staging, pré-visualização, confirmação e auditoria antes de alterar a base da folha.
