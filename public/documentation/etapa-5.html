Etapa 5 — Rubricas e eventos

Mapeamento para importações e folha · ETA: 15–25 min

URL base: #/wizard/etapa-5/ · Código: frontend/src/pages/wizard/stage5.tsx

Objetivo: cadastrar códigos de proventos/descontos/informativos, marcar incidências trabalhistas e eSocial, e opcionalmente definir centro de custo e departamento padrão — alinhando com a pré-visualização de lotes eSocial na etapa 6.

Índice de passos

  • passo-1 Lista e cadastro manual
  • passo-2 Incidências e padrões CC/depto
  • passo-3 Planilha + importação CSV em textarea
  • resumo Checklist e envio

passo-1 — Lista e cadastro

Tabela com todas as rubricas; modal para criar nova.

Campo Obrig. Para que serve Como funciona
code Sim Código curto único (holerite, integrações). POST /payroll-rubrics; trim no envio.
name Sim Descrição legível do evento. —
rubricType Sim Natureza do lançamento. EARNING (Provento), DEDUCTION (Desconto), INFORMATIVE (Informativo).

Colunas da tabela: Código, Nome, Tipo.

passo-2 — Incidências e regras

Por rubrica, botão “Incidências / CC” abre modal de edição parcial (não altera código/nome/tipo aqui).

Campo Obrig. Para que serve Como funciona
incidenceINSS Não Incide contribuição previdenciária. Checkbox → boolean no PUT.
incidenceFGTS Não Base de FGTS. Checkbox.
incidenceIRRF Não Impacta IRRF. Checkbox.
incidenceESocial Não Envio/monitoração eSocial. Checkbox.
defaultCostCenterId Não CC sugerido quando o lançamento não trouxer centro. Select pesquisável das CC da etapa 2; pode limpar.
defaultDepartmentId Não Departamento sugerido. Select pesquisável dos departamentos da etapa 2.

PUT /payroll-rubrics/:id reenvia também code, name e rubricType da linha em edição.

passo-3 — Importação planilha

Elemento Para que serve Como funciona
Upload de arquivo Arquivar planilha para auditoria. Upload categoria RUBRICS_SPREADSHEET (não parseia automaticamente o arquivo no front).
Textarea CSV Criar várias rubricas de uma vez. Uma linha por rubrica: codigo;nome;tipo. O terceiro campo é enviado em maiúsculas ao backend; use exatamente EARNING, DEDUCTION ou INFORMATIVE (conforme placeholder da tela — “INFORMATIVO” sozinho não é mapeado automaticamente).

Botão “Importar linhas CSV” → POST /payroll-rubrics/import com array rows; resposta informa criadas e códigos ignorados (duplicados).

resumo

  • Rubricas cadastradas ou importadas (contagem)
  • Lembrete para revisar incidências e CC/depto (passos 1–2)

Enviar etapa 5 → POST /wizard/stages/5/submit. Há atalho para etapa 6.

Rubricas mapeadas alimentam alertas na etapa 6 (ex.: eventos não mapeados na pré-visualização do lote eSocial).
