Jornada do cliente

Antigo “Fluxos do usuário” — mesma informação, título pensado para quem apresenta para negócio.

Leitura rápida para diretoria: a ordem abaixo é o caminho feliz — login, painel, depois o wizard das seis etapas. Na prática, o consultor pode pausar entre etapas para validar dados; o importante é que nada é “mágico”: cada bloco tem lugar na sequência. Ver narrativa executiva completa →

Visão linear de como as telas se encaixam na jornada de parametrização. A ordem abaixo é a recomendada pelo produto; o backend e o time consultivo podem exigir aprovações entre etapas.

1. Acesso e painel

Login (#/login)
    → Dashboard (#/dashboard)
    → (opcional) Configurações (#/settings), Uploads (#/uploads)

2. Wizard completo (macrofluxo)

Hub (#/wizard)
    → Etapa 1 … Etapa 6 (cada uma: passo-1 → … → resumo ou passo final)
    → Envio / validação por etapa conforme botões na UI
Dependência lógica (negócio):
  1. Empresa e filiais definidas antes de estrutura operacional consistente.
  2. Estrutura (etapa 2) usada em cadastros e rubricas nas etapas seguintes.
  3. Base histórica (3) e benefícios/rubricas (4–5) alimentam conferências na importação eSocial (6).

3. Fluxo dentro de uma etapa

  1. Usuário abre #/wizard/etapa-N (vai para passo-1).
  2. Navega por passos via menu do WizardStageLayout ou ações na própria página.
  3. No resumo (etapas 1–5), revisa checklist e dispara envio para validação quando disponível.
  4. Etapa 6 encerra em status do lote; lotes específicos podem abrir em …/lote/:batchId.

4. Erros e rotas desconhecidas

URL autenticada inválida → NotFoundPage. Sessão expirada → fluxo típico do ProtectedRoute (ex.: retorno ao login).
