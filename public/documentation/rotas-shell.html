Rotas e shell da aplicação Apêndice técnico

Página para desenvolvimento. Narrativa de negócio: O que vamos entregar.

O arquivo App.tsx define o HashRouter. A raiz autenticada redireciona para /dashboard. O layout com menu lateral é o AppShell.

Rotas públicas

  • #/login — LoginPage
  • #/documentacao/diagrama-banco — DatabaseErDiagramPage (diagrama ER Mermaid + documentação tabular do schema; não exige login). Também em /documentation/diagrama-banco.html (HTML estático com iframe para essa rota).
  • /documentation/swagger.html (estático em public/documentation) — UI OpenAPI do backend em iframe + link em nova aba; fora do HashRouter

Rotas protegidas (AppShell)

  • #/dashboard — painel inicial
  • #/settings — configurações da conta / contexto
  • #/uploads — gestão de uploads
  • #/wizard — hub do wizard (lista de etapas)
  • #/wizard/etapa-<1–6>/… — passos do wizard (ver páginas por etapa)
  • #/wizard/etapa-6/lote/:batchId — detalhe de lote eSocial
  • Qualquer outra rota autenticada desconhecida → NotFoundPage

Wizard: padrão de URL

Cada etapa usa o layout WizardStageLayout e um stepSlug na URL:

#/wizard/etapa-2/passo-3
#/wizard/etapa-2/resumo

O índice de cada etapa (#/wizard/etapa-N) redireciona para passo-1. A etapa 6 tem rota extra lote/:batchId antes do fallback :stepSlug.

Componentes de rota (wizard)

  • StepOutlet — lê etapa e stepSlug dos params e renderiza StepContent.
  • StepContent — escolhe stage1 … stage6 conforme o número da etapa.

O mapeamento de slug para progresso no backend está em slugToBackendStepNumber (utils/wizardSteps.ts) usando STAGE_META.
